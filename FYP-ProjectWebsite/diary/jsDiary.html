Reflective Diary

Jia Sheng's Reflection Journal

Week 1

12 April 2026 - 19 April 2026

Project Direction

Work Done

  • Presented the EverMotion project idea to Mr Tian.
  • Discussed the need to avoid keyboard and mouse interaction.
  • Brainstormed rehabilitation game ideas for grip strength, steady hand control, and cognitive engagement.
  • Contacted St Luke's Eldercare HQ at SIM to explore collaboration opportunities.

Learning

  • MediaPipe can detect 21 hand landmarks using a standard webcam.
  • OpenCV can support webcam input, preprocessing, and tracking visual overlays.
  • Calibration gestures can help adapt the system to each user's movement range.

Challenges

  • Hand tracking may be affected by tremors, restricted movement, or poor lighting.
  • The system may need smoothing algorithms and better error handling to stay usable for elderly users.

Reflection

  • This week helped confirm that calibration is central to EverMotion.
  • The game should adapt interaction areas, object positions, and speed based on each user's movement ability.

Next Steps

  • Continue refining the calibration approach.
  • Seek real-world feedback from eldercare or rehabilitation partners to ensure the games provide meaningful therapeutic value.
Week 2

19 April 2026 - 26 April 2026

Scope Expansion

Work Done

  • Helped the team re-scope the project after feedback that three mini-games were insufficient for the team size.
  • Facilitated discussion and helped synthesize ideas for 12 rehabilitation games across shoulder, elbow, wrist, and fingers.
  • Proposed the Fruit Harvest concept, which became the creative foundation for the Side Stretch game under the shoulder category.
  • Built a simple Fruit Harvest prototype using MediaPipe and Pygame to demonstrate real-time hand tracking to the team.

Learning

  • MediaPipe hand landmark tracking works in real time, but latency can become noticeable under certain conditions.
  • Tracking accuracy drops when the hand moves near the edges of the webcam frame.
  • These limitations affect game boundary design and reinforce the need for proper calibration.

Reflection

  • The project scope changed significantly this week, but the 12-game direction creates a stronger rehabilitation platform.
  • Facilitating the brainstorming session helped keep the discussion focused while ensuring each member's ideas were heard.
  • Building the prototype was valuable because practical testing revealed latency and tracking-range risks that documentation alone did not show clearly.
Week 3

26 April 2026 - 03 May 2026

Website Deployment

Work Done

  • Built the first version of the EverMotion project website with the team overview, project description, and revised 12-game structure.
  • Presented the website to teammates, collected feedback, and refined the layout and content before external access.
  • Collaborated with Jun Hui to add administrative materials such as meeting minutes and progress tracking.
  • Worked with Gavin to deploy the website through Vercel and connect it to the team GitHub repository.

Learning

  • Vercel integrates with GitHub so pushes to the repository can automatically trigger deployment.
  • Each deployment can generate a preview URL, which is useful for teammate review before finalising changes.
  • The production URL remains stable, giving supervisors and assessors a permanent link to monitor progress.
  • Continuous Integration should be explored next to support the Continuous Delivery setup already in place.

Reflection

  • Building and deploying the website was more involved than expected, but it gave the project a more professional and accessible presence.
  • Working with Gavin on Vercel made the Continuous Delivery process clear through the GitHub push-to-deploy workflow.
  • The work highlighted a gap in integration checks, making CI an important next step as more members contribute code.

Stakeholder Communication

  • Working with Jun Hui on administrative content showed that the website is also a communication tool for stakeholders.
  • The website allows Mr Tian and Mr Ee Kiam Keong to check project progress without waiting for scheduled updates.
