Week 1

12 April 2026 - 18 April 2026

Project Direction

Work Done

  • Visited an elderly care or rehabilitation-related facility and spoke with staff about common rehabilitation challenges.
  • Learned that elderly users may lose motivation when exercises become repetitive.
  • Brainstormed healthcare, rehabilitation, and elderly support project themes with the team.
  • Helped prepare initial project proposal ideas, including background, objectives, scope, and possible tools.

Learning

  • Accessibility and user engagement are important when designing for elderly users.
  • The team should focus on simple, accessible, and engaging rehabilitation activities.
  • Possible features such as webcam tracking, calibration, and mini-games helped shape the early project scope.

Challenges

  • The project idea was still broad and needed further refinement.
  • It was difficult to decide which features were realistic within the project timeframe.

Reflection

  • The first week helped the team understand the needs of elderly users and people with motor impairments.
  • The visit and discussions supported the decision to focus on EverMotion as a motion-based rehabilitation game using webcam tracking.
Week 2

19 April 2026 - 25 April 2026

Requirements

Work Done

  • Created user stories for elderly users, caregivers, therapists, and rehabilitation staff.
  • Wrote user stories for login, calibration, Story Mode, saving progress, and mini-games.
  • Helped set up the basic project website structure with project title, team details, objectives, and description.
  • Worked on proposal content such as background, objectives, stakeholders, requirements, tools, and Agile methodology.

Learning

  • User stories helped the team think from the users' point of view instead of only focusing on technical functions.
  • The project website helped present the team's idea and progress in a more organized way.
  • Development tools discussed included Unity, MediaPipe, Python, C#, and MySQL.

Challenges

  • Some user stories were difficult to write clearly at first.
  • The team had to decide which features were most important for the proposal.

Reflection

  • The second week helped organize the project idea into clearer requirements.
  • Thinking through different user interactions made the system requirements more practical.
Week 3

26 April 2026 - 02 May 2026

Visual Design

Work Done

  • Designed early concepts for the main character and robot avatar for Story Mode.
  • Planned the robot's appearance and energy restoration progress.
  • Discussed clean background styles and basic UI elements such as buttons, menus, tutorials, and progress bars.
  • Planned how sprites and UI assets could be integrated into Unity with visual feedback for mini-games and the 12-bar energy system.

Learning

  • Visual design should be attractive while remaining simple and accessible for elderly users.
  • Readable text, clear instructions, simple backgrounds, and friendly character design can make the game more comfortable and engaging.

Challenges

  • It was difficult to balance attractive visuals with simple and accessible design.
  • Some UI elements still need further adjustment before implementation.

Reflection

  • The third week showed that EverMotion's design should not only look good, but also be easy for elderly users to understand.
  • Clear UI and friendly visuals are important for making the rehabilitation game more engaging.
