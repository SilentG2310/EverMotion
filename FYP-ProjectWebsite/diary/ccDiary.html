Reflective Diary

Channelle's Reflection Journal

Week 1

12 April 2026 - 18 April 2026

Project Direction

Work Done

  • Joined the team and participated in early discussions to decide the project direction.
  • Contributed ideas for the game design and possible gameplay mechanics suitable for the target users.
  • Started working on the project proposal with the team by understanding the requirements and structure.

Learning

  • Learned that a good proposal needs to present the project concept clearly and logically.
  • Understood that game ideas must be considered from both creativity and feasibility perspectives.

Challenges

  • Found it challenging to translate early ideas into concrete gameplay logic.
  • Needed to ensure the proposed mechanics were practical and suitable for development.

Reflection

  • This week helped the team move towards a clearer project concept.
  • I learned to think more critically about feasibility instead of focusing only on creative ideas.
Week 2

19 April 2026 - 25 April 2026

Research

Work Done

  • Continued updating and refining different sections of the project proposal with the team.
  • Took part in research by exploring potential rehabilitation centres and old folks homes with the team.
  • Helped the team better understand possible needs of elderly users.

Learning

  • Realised that gathering real-world insights can be more difficult than expected.
  • Gained a better understanding of exercises and interactions that may be suitable for elderly users.

Challenges

  • Access to rehabilitation centres and elderly care facilities was not always straightforward.
  • Direct feedback from actual users was limited due to restricted access.

Reflection

  • I adapted by relying more on discussions, observations, and available information to support design decisions.
  • This helped me develop a more flexible approach towards research.
Week 3

26 April 2026 - 02 May 2026

Documentation & UI Design

Work Done

  • Focused on the system design aspect of the project.
  • Worked on user stories and contributed to creating diagrams for the system.
  • Continued supporting updates to the project proposal to keep the content aligned and clear.

Learning

  • Learned to think from the user's perspective when designing system features.
  • Understood that diagrams and user stories help translate ideas into a more structured format.

Challenges

  • Ensuring the user stories and diagrams accurately reflected the intended system behaviour required several revisions.
  • It was important to make sure the system components were logically connected.

Reflection

  • This week showed me the importance of attention to detail in system design.
  • I learned that clear thinking is needed when turning project ideas into system components.
Week 4

03 May 2026 - 10 May 2026

UI Design

Work Done

  • Worked with team to complete PRD for submission by finalizing our user stories, non-functional requirements, including use case diagrams
  • Continue to develop visual assets for our games

Challenges

  • Continued revising the document by refining user stories to better reflect the game's functionality and user needs before the submission deadline
  • Focus on designing simple yet visually appealing UI and graphics suitable for rehabilitation purposes, while considering smooth motion-based interactions and user-friendly screen transitions for elderly users

Reflection

  • This week underscored the importance of iteration in both documentation and design.
  • Developed greater flexibility and persistence when improving project deliverables, while adapting to feedback and new ideas within a collaborative team environment
  • Gained a better understanding of how technical development and creative design must work together to create a cohesive and engaging user experience
Week 7

24 May 2026 - 30 May 2026

Documentation & UI Design

Work Done

  • Continued Refinement of URS and TDM
  • Development of Additional System Diagrams
  • Supporting Game Development and Asset Integration

Challenges

  • Keeping Documentation Up-to-Date as the diagrams and documentation need to update frequently to ensure that it is accurately reflected the latest version of the system
  • Learning Development Tools and Workflow

Reflection

  • This week was particularly rewarding because I was able to contribute not only to the documentation side of the project but also to the actual game development process
  • Improved understanding of how individual features interact within the overall system through the creation of activity and sequence diagrams
