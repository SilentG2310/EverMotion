About

The Problem We're Solving

⚠

The Problem

Traditional rehabilitation exercises are repetitive and unengaging, causing users — especially elderly individuals and patients with motor impairments — to lose motivation and participate inconsistently.

◆

Our Solution

EverMotion allows users to perform rehabilitation exercises through natural hand movements using a standard webcam. By combining AI hand tracking with game mechanics, therapy becomes an engaging experience.

Why It Matters

📈 Improves motivation in rehabilitation
👤 Encourages consistent participation
💻 No specialised equipment needed
🧠 Accessible for elderly & motor-impaired users

Features

What EverMotion Offers

📷

Real-Time Hand Tracking

Uses MediaPipe via standard webcam — no gloves, controllers, or extra hardware needed.

👤

User Profiles & Avatars

Personalised accounts with avatar selection so each user has their own experience.

🎯

Hand Calibration

A short calibration flow adapts the game to each user's movement range and interaction ability.

📈

Adaptive Difficulty

Game elements like speed and positioning adjust automatically to suit the player's ability.

Technology

Under the Hood

Game Engine
Unity

Handles gameplay logic, UI, animations, and real-time rendering.

AI / Hand Tracking
MediaPipe

Real-time hand landmark detection from webcam input — no extra hardware required.

Backend / AI Processing
Python

Runs the MediaPipe pipeline, handles gesture processing and future model optimisation.

Game Programming
C#

Controls player movement, input handling, and core game logic inside Unity.

Communication
UDP

Low-latency real-time data transfer from Python to Unity — ideal for games.

3D & Animation
Blender

Character modelling, animation creation, and asset export into Unity.

Project Management
Taiga & GitHub

Scrum sprint tracking via Taiga; continuous integration via GitHub.

Database
MySQL

Stores user profiles, calibration data, and session statistics.
